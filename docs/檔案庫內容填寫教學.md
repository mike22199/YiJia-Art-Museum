# 義家藝館檔案庫 — 各頁面內容填寫教學

> 適用日期：2026-06  
> 大部分內容只需編輯 **`content/site-content.json`**，圖片放入 **`assets/`** 資料夾，**通常不必改程式**。

---

## 一、開始之前

### 1.1 如何預覽網站

1. 用 VS Code **Live Server**，或任何本機靜態伺服器，開啟專案根目錄的 `index.html`。
2. 網址列加上 `#` 後的路由即可進入各頁，例如：  
   `http://localhost:5500/index.html#archive/index`

### 1.2 主要編輯檔案

| 檔案 | 用途 |
|------|------|
| `content/site-content.json` | 全站文字、圖片路徑、連結（**最主要**） |
| `assets/images/…` | 各頁面用的圖片 |
| `assets/journals/…` | 教師日誌逐頁圖（選用） |

### 1.3 全站共用設定（每頁都會出現）

這些在 JSON **最上方**，不在 `archive` 裡：

| 區塊 | JSON 路徑 | 要補什麼 |
|------|-----------|----------|
| 網站標題 | `site.title`、`site.subtitle` | 站名、英文副標 |
| Header Logo | `site.headerLogo` | 小 house 圖示路徑 |
| 頁尾 | `site.footer` | 地址、電話、傳真、Email |
| 社群連結 | `site.footer.social` | YouTube、Facebook 網址 |
| 子導覽選單 | `archiveSubnav[]` | 各子頁名稱與連結（通常不用改） |

**子導覽預設順序：** 檔案庫首頁 → 歷屆展覽回顧 → 藝術實踐 → 影音 → 典藏 → 研究 → 藝術家教師日誌 → 參考書目

若要改選單文字或順序，編輯 `archiveSubnav` 陣列中每一項的 `label` 與 `href` 即可。

## 三、建議素材資料夾

```
assets/
  images/
    archive/              ← 檔案庫首頁 Banner
    exhibitions-past/     ← 歷屆展覽海報、作品照
    practice/             ← 藝術實踐作品照
    media/                ← 影音輪播、田野踏查照片
    collection/           ← 典藏作品主圖、細部圖
    teachers/2026/        ← 教師頭像
    books/                ← 參考書目書封
  journals/
    2026/t01/             ← 教師日誌逐頁 JPG（page01.jpg, page02.jpg…）
```

圖片放好後，JSON 裡把 `"src"` 改成對應路徑，例如：  
`"./assets/images/exhibitions-past/2026-work01.jpg"`

**圖片建議：**
- Banner／海報：寬至少 1200px
- 作品縮圖：建議 4:3 或 1:1
- 教師頭像：正方形，至少 200×200px
- 書封：直式，長邊至少 600px
- 日誌：PDF 請先轉成逐頁 JPG 再填入

## 四、各頁面詳細說明

### 4.1 檔案庫首頁

**預覽：** `#archive/index`  
**JSON：** `archive.home`

**頁面由上而下：**

```
Header → 子導覽 → 主視覺 Banner → 義家藝館介紹（作品／藝術家教師按鈕）
→ 特展 → 精選影音 → 歷屆展覽回顧預覽 → Footer
```

| 畫面區塊 | JSON 路徑 | 要提供的內容 |
|----------|-----------|--------------|
| 主視覺 Banner | `home.banner` | 大圖 `image.src`、`title`、`subtitle` |
| 義家藝館介紹 | `home.intro.heading`、`home.intro.body` | 機構介紹標題與正文 |
| 「作品」「藝術家教師」按鈕 | `home.intro.links[]` | 每項：`label`（按鈕文字）、`href`（連結） |
| 特展 | `home.specialExhibitions[]` | 特展名稱、縮圖、連結（預設連到兩個開幕展） |
| 精選影音 | `home.featuredMedia[]` | 每則：標題、說明、縮圖 |
| 精選影音「了解更多」 | `home.featuredMediaMoreHref` | 通常設 `#archive/media` |
| 歷屆展覽預覽 | `home.pastExhibitionsPreview.items[]` | 作品名、作者、縮圖（建議 4 則） |

**intro.links 範例：**

```json
"links": [
  { "label": "作品", "href": "#archive/practice" },
  { "label": "藝術家教師", "href": "#archive/teachers" }
]
```

**常見調整：**
- 改 Banner 圖：只改 `home.banner.image.src`
- 「作品」要連到歷屆展覽：把 `href` 改成 `#archive/exhibitions`
- 特展圖與連結：編輯 `specialExhibitions` 陣列

### 4.2 歷屆展覽回顧

**預覽：** `#archive/exhibitions?year=2026`  
**JSON：** `archive.exhibitionYears`、`archive.exhibitionsByYear`

**頁面由上而下：**

```
Header → 子導覽 → 標題 → 年份 Tab
→ 海報｜展覽概論 → 網站／影音／藝術家教師 按鈕
→ 作品網格 → 藝術家教師頭像網格 → Footer
```

| 畫面區塊 | JSON 路徑 | 要提供的內容 |
|----------|-----------|--------------|
| 年份 Tab | `exhibitionYears[]` | 要顯示的年份列表，如 `["2016","2018",…,"2026"]` |
| 海報 | `exhibitionsByYear["2026"].poster` | 該年展覽海報圖 |
| 展覽概論 | `…overview.heading`、`…overview.body` | 該年展覽概述文字 |
| 快捷按鈕 | `…quickLinks[]` | 網站、影音、藝術家教師的連結（可省略，有預設值） |
| 作品 | `…works[]` | 作品名、作者、作品照 |
| 藝術家教師 | `…teachers[]` | 頭像、姓名（**可省略**，會自動從 `archive.teachers.byYear` 帶入） |

**單一年份完整範例：**

```json
"2026": {
  "poster": { "src": "./assets/images/exhibitions-past/2026-poster.jpg", "alt": "2026 海報" },
  "overview": {
    "heading": "展覽概論",
    "body": "2026 年度展覽正式介紹文字……"
  },
  "quickLinks": [
    { "label": "網站", "href": "https://你的年度網站網址" },
    { "label": "影音", "href": "#archive/media?year=2026" },
    { "label": "藝術家教師", "href": "#archive/teachers?year=2026" }
  ],
  "works": [
    {
      "title": "作品名稱",
      "author": "作者姓名",
      "image": { "src": "./assets/images/exhibitions-past/2026-01.jpg", "alt": "" },
      "collectionId": "ex-2026-01"
    }
  ]
}
```

**作品方塊 → 典藏詳情頁：** 每件作品加上 `collectionId`，並在 `archive.collection.items` 建立同 id 的典藏資料（詳見 **§4.7**）。有 `collectionId` 時，作品方塊可點擊，會進入 `#archive/collection?work=ex-2026-01`。

**常見調整：**
- 新增年份：在 `exhibitionYears` 加年份，並新增同名 key 的物件
- 「網站」要連外部網址：`quickLinks` 第一項填完整 `https://…`
- 教師名單與教師頁同步：在 `archive.teachers.byYear["2026"].teachers` 維護即可
- 新增展覽作品並連結典藏：見 **§4.7「如何新增並連結作品」**

### 4.3 藝術實踐

**預覽：** `#archive/practice?year=2025`  
**JSON：** `archive.practice`

**頁面由上而下：**

```
Header → 子導覽 → 年份選擇｜作者選擇｜關鍵字搜尋 → 作品三欄網格 → Footer
```

| 畫面區塊 | JSON 路徑 | 要提供的內容 |
|----------|-----------|--------------|
| 年份選單 | `practice.filterYears[]` | 如 `["2024","2025"]` |
| 作者選單 | `practice.authors[]` | 所有作者姓名（供篩選） |
| 作品 | `practice.worksByYear["2025"][]` | 作品名、作者、作品照 |

**單一作品格式：**

```json
{
  "title": "作品名稱",
  "author": "作者姓名",
  "image": { "src": "./assets/images/practice/2025-work01.jpg", "alt": "" },
  "collectionId": "pr-2025-01"
}
```

**作品方塊 → 典藏詳情頁：** 與歷屆展覽相同，透過 `collectionId` 連到典藏（詳見 **§4.7**）。

**常見調整：**
- 新增年份：在 `filterYears` 加年份，並新增 `worksByYear["新年份"]` 陣列
- 作者篩選：`authors` 裡的姓名必須與作品的 `author` **完全一致**才篩得到
- 關鍵字搜尋：會搜尋作品名與作者，不需額外設定
- 新增實踐作品並連結典藏：見 **§4.7「如何新增並連結作品」**

### 4.4 影音（含照片與影片）

**預覽：** `#archive/media?year=2025`  
**JSON：** `archive.media`

**頁面由上而下：**

```
Header → 子導覽 → 精選照片輪播 → 年份｜關鍵字搜尋
→ 「2025田野踏查紀錄」→ 左側照片／影片網格 ＋ 右側概述／圖片說明 → Footer
```

| 畫面區塊 | JSON 路徑 | 要提供的內容 |
|----------|-----------|--------------|
| 精選輪播 | `media.featuredCarousel[]` | 輪播用大圖、標題、說明 |
| 年份選單 | `media.years[]` | 如 `["2025"]` |
| 區塊標題 | `media.byYear["2025"].title` | 如 `"2025田野踏查紀錄"` |
| 田野踏查概述 | `…overview` | 右側紅色區的年度概述文字 |
| 照片 | `…items[]`（`type: "photo"`） | 縮圖、`caption`（點選後顯示在黃色區） |
| 影片 | `…items[]`（`type: "video"`） | 縮圖、`caption`、`youtubeUrl` |

**照片項目：**

```json
{
  "type": "photo",
  "title": "踏查照片一",
  "caption": "這張照片的說明文字",
  "image": { "src": "./assets/images/media/2025-01.jpg", "alt": "" }
}
```

**影片項目（點擊會開 YouTube）：**

```json
{
  "type": "video",
  "title": "踏查紀錄影片",
  "caption": "影片說明",
  "image": { "src": "./assets/images/media/2025-video-thumb.jpg", "alt": "" },
  "youtubeUrl": "https://www.youtube.com/watch?v=影片ID"
}
```

**常見調整：**
- 輪播與下方網格可放不同圖：`featuredCarousel` 管上方大輪播，`byYear.items` 管下方網格
- 新增年份：在 `years` 加年份，並新增 `byYear["新年份"]` 物件
- 右側「圖片說明」：來自各項目的 `caption`，使用者點縮圖時更新

### 4.5 藝術家教師日誌

**預覽：** `#archive/teachers?year=2026&teacher=t01`  
**JSON：** `archive.teachers`

**頁面由上而下：**

```
Header → 子導覽 → 藝術家教師日誌
→ 左：頭像／姓名／簡介 ｜ 右：ISSUU 式雙頁翻書（可放大）
→ 2026年藝術家教師 頭像輪播 → 年份時間軸 → Footer
```

| 畫面區塊 | JSON 路徑 | 要提供的內容 |
|----------|-----------|--------------|
| 年份時間軸 | `teachers.years[]`、`teachers.defaultYear` | 2019～2026 等 |
| 該年教師名單 | `teachers.byYear["2026"].teachers[]` | `id`、`name`、`bio`、`avatar` |
| 教師日誌 | `teachers.journals["t01"].pages[]` | 日誌每一頁的內容 |

**教師格式：**

```json
{
  "id": "t01",
  "name": "王小明",
  "bio": "教師簡介文字（100～200 字）",
  "avatar": { "src": "./assets/images/teachers/2026/t01.jpg", "alt": "" }
}
```

**日誌 — 文字版（暫用）：**

```json
"journals": {
  "t01": {
    "pages": [
      { "title": "第一頁", "body": "日誌內容……", "color": "#FFE4E1" },
      { "title": "第二頁", "body": "日誌內容……", "color": "#FFF8DC" }
    ]
  }
}
```

**日誌 — 圖片版（正式素材，較接近 ISSUU）：**

```json
"pages": [
  {
    "image": {
      "src": "./assets/journals/2026/t01/page01.jpg",
      "alt": "日誌第 1 頁"
    }
  },
  {
    "image": {
      "src": "./assets/journals/2026/t01/page02.jpg",
      "alt": "日誌第 2 頁"
    }
  }
]
```

**重要規則：**
- 教師 `id`（如 `t01`）必須與 `journals` 的 key **完全一致**
- 2026 年預計 9 位教師：`t01`～`t09`
- 翻書操作：點書本左半 → 上一頁；右半 → 下一頁；或按下方按鈕；「放大檢視」可放大閱讀

**常見調整：**
- 切換預設年份：改 `defaultYear`
- 新增某年教師：在 `byYear` 加 `"2018": { "title": "2018年藝術家教師", "teachers": [...] }`，並在 `years` 陣列加入 `"2018"`
- PDF 日誌：用 PDF 轉 JPG 工具逐頁匯出，依序命名 page01.jpg、page02.jpg…

### 4.7 典藏與作品連結（歷屆展覽・藝術實踐・典藏）

> **核心概念：** 「典藏詳情頁」是每件作品的**完整介紹頁**。歷屆展覽、藝術實踐列表上的作品方塊，以及典藏列表，都透過同一組 `id` 連到這個頁面。

#### 三個頁面如何串在一起

```
歷屆展覽 #archive/exhibitions?year=2026
  └─ 作品方塊（collectionId: ex-2026-01）
        ↓ 點擊
典藏詳情 #archive/collection?work=ex-2026-01
        ↑ 點擊
藝術實踐 #archive/practice?year=2025
  └─ 作品方塊（collectionId: pr-2025-01）
```

| 頁面 | 預覽網址 | JSON 區塊 | 作品上的連結欄位 |
|------|----------|-----------|------------------|
| 歷屆展覽回顧 | `#archive/exhibitions?year=2026` | `exhibitionsByYear["2026"].works[]` | `collectionId` |
| 藝術實踐 | `#archive/practice?year=2025` | `practice.worksByYear["2025"][]` | `collectionId` |
| 典藏列表 | `#archive/collection` | `collection.items[]` | `id`（與上方 `collectionId` 相同） |
| 典藏詳情 | `#archive/collection?work=ex-2026-01` | `collection.items[]` 中 `id` 相符的那一筆 | — |

**連結規則（最重要）：**

1. 歷屆展覽／藝術實踐的每件作品，加上 **`collectionId`**
2. `archive.collection.items` 裡，建立 **`id` 與 `collectionId` 完全相同** 的一筆典藏資料
3. 兩邊的 `title`、`author`、`image` 建議保持一致；典藏頁可再補 `body`、`materials` 等詳細欄位

**建議 id 命名：**

| 來源 | 格式 | 範例 |
|------|------|------|
| 歷屆展覽 | `ex-{年份}-{序號}` | `ex-2016-01`、`ex-2026-04` |
| 藝術實踐 | `pr-{年份}-{序號}` | `pr-2025-01`、`pr-2025-06` |

#### 典藏詳情頁內容

點選任一作品方塊後，網址為 `#archive/collection?work=作品id`，頁面顯示：

- 主圖、作品名、作者
- 選填：年代、媒材、尺寸、典藏編號
- 正文 `body`、細部圖 `images`
- 導覽：「返回典藏列表」＋「返回歷屆展覽回顧 2016」或「返回藝術實踐 2025」（來自 `source` 欄位）

#### 典藏 `items[]` 欄位說明

| 欄位 | 必填 | 說明 |
|------|------|------|
| `id` | ✓ | 唯一代碼，與展覽／實踐作品的 `collectionId` 相同 |
| `title` | ✓ | 作品名稱 |
| `author` | ✓ | 作者 |
| `image` | ✓ | 主圖 `{ "src": "...", "alt": "" }` |
| `year` | | 年份 |
| `body` | | 典藏說明正文（`\n` 可換行） |
| `materials` | | 媒材 |
| `dimensions` | | 尺寸 |
| `accessionNo` | | 典藏編號 |
| `images` | | 細部圖 `[{ "src": "...", "alt": "" }]` |
| `source` | | 返回來源頁（系統用，見下方範例） |

**歷屆展覽作品的完整範例（兩處都要寫）：**

`exhibitionsByYear["2016"].works[]` 其中一筆：

```json
{
  "title": "2016 作品一",
  "author": "作者",
  "image": { "src": "./assets/images/exhibitions-past/2016-01.jpg", "alt": "" },
  "collectionId": "ex-2016-01"
}
```

`collection.items[]` 對應一筆：

```json
{
  "id": "ex-2016-01",
  "title": "2016 作品一",
  "author": "作者",
  "year": "2016",
  "materials": "複合媒材",
  "dimensions": "30 × 40 cm",
  "accessionNo": "FIFI-2016-001",
  "image": { "src": "./assets/images/exhibitions-past/2016-01.jpg", "alt": "" },
  "body": "典藏說明正文。可描述創作背景、保存狀況與展覽經歷。",
  "source": {
    "type": "exhibition",
    "year": "2016",
    "label": "歷屆展覽回顧 2016",
    "href": "#archive/exhibitions?year=2016"
  }
}
```

**藝術實踐作品** 同理，`collectionId` 用 `pr-2025-01` 格式，`source.type` 改為 `"practice"`，`href` 改為 `#archive/practice?year=2025`。

#### 如何新增並連結作品（手動步驟）

**情境 A：在歷屆展覽新增一件作品**

1. 圖片放入 `assets/images/exhibitions-past/`
2. 在 `exhibitionsByYear["年份"].works` 新增物件，設好 `collectionId`（如 `ex-2026-05`）
3. 在 `archive.collection.items` 新增 `id` 相同的典藏物件，並填 `body`、`source` 等
4. 預覽：
   - 展覽列表：`#archive/exhibitions?year=2026` → 點作品方塊
   - 典藏詳情：`#archive/collection?work=ex-2026-05`
   - 典藏列表：`#archive/collection`

**情境 B：在藝術實踐新增一件作品**

1. 圖片放入 `assets/images/practice/`
2. 在 `practice.worksByYear["年份"]` 新增物件，設 `collectionId`（如 `pr-2025-07`）
3. 在 `collection.items` 新增對應典藏資料
4. 若為新作者，記得同步更新 `practice.authors[]`

**情境 C：只出現在典藏列表、不在展覽／實踐頁**

1. 只在 `collection.items` 新增一筆（設 `id`，不必寫 `collectionId` 到別處）
2. 可省略 `source`，詳情頁只會顯示「返回典藏列表」

#### 批次同步（開發協助用）

若已有一批展覽／實踐作品但尚未建立典藏資料，可執行：

```bash
node studio/scripts/sync-collection-links.mjs
```

此腳本會自動為每件作品產生 `collectionId`，並寫入 `collection.items`（含 `source` 返回連結）。**執行後請再手動補上各作品的 `body`、媒材等正式內容。**

#### 常見調整

- **作品方塊不能點？** 檢查是否有 `collectionId`，且 `collection.items` 裡有相同 `id`
- **點了顯示找不到？** `collectionId` 與典藏 `id` 不一致，或典藏陣列漏了該筆
- **改 id：** 展覽／實踐的 `collectionId` 與典藏 `id` 要一起改
- **刪除作品：** 三處都要刪（展覽或實踐 `works`、`collection.items`）

## 七、常見問題

### Q1：改了 JSON 但網頁沒變？
- 重新整理瀏覽器（Ctrl+F5 強制刷新）
- 確認改的是 `content/site-content.json`，不是其他檔案
- 確認 JSON 語法正確（逗號、引號）；可用 [jsonlint.com](https://jsonlint.com) 檢查

### Q2：圖片顯示不出來？
- 路徑以 `./` 開頭，相對於 `index.html`
- 檔名大小寫需一致（Windows 上較寬鬆，上線後 Linux 會區分）
- 確認圖片已放入 `assets/` 對應資料夾

### Q3：想改按鈕連結、不要改程式？
- 首頁「作品／藝術家教師」→ `archive.home.intro.links`
- 歷屆展覽「網站／影音／藝術家教師」→ 各年份的 `quickLinks`
- 子導覽 → `archiveSubnav`

### Q4：想改版型、顏色、動畫？
- 需改程式：`app/archive-pages.js`（結構）、`app/styles.css`（樣式）
- 純換內容不必動這兩個檔案

### Q5：之後要用 Sanity CMS 管理？
- 請參考同資料夾的 `SANITY_使用教學.md`
- 欄位結構應與本文件 JSON 路徑對應

### Q6：歷屆展覽、藝術實踐的作品怎麼連到典藏頁？
- 在展覽／實踐作品的 JSON 加 **`collectionId`**（如 `ex-2026-01`）
- 在 `archive.collection.items` 建立 **`id` 相同** 的典藏資料
- 有設定且典藏有對應資料時，作品方塊可點擊，進入 `#archive/collection?work=ex-2026-01`
- 詳情頁可顯示完整 `body`，並提供返回展覽／實踐列表的連結（`source` 欄位）
- 完整教學見 **§4.7 典藏與作品連結**
